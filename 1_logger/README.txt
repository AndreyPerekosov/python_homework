Скрипт log_analyzer.py обрабатывает при запуске последний (c наиболее поздней датой в имени) лог в формата nginx, 
в результате работы получаеся html-отчет с аналогичной датой вида report-%Y.%m.%d.html
Формирование отчета происходит на основе шаблона report.html.
Запуск из директории со скриптом:  python log_analyzer.py 
Рядом со скриптом находится файл settings.txt. В данном файле можно можно определить базовые настройки в виде {"КЛЮЧ1": "строковое_значение", "КЛЮЧ2": цифровое_значение}

Доступные настройки:
    key                        description				default				
"REPORT_SIZE"   количество записей из обрабатываемого лога в отчете	  1000
"LOG_DIR"  		путь к папку с обработанными логами    		  "./log"
"REPORT_DIR"		    путь к папке  				  "./report"	
"LOG_FILE"		путь до лог-файла работы скрипта                   None
"DEFAULT_CONFIG": 	  путь до файла настроек			"setting.txt"

При запуске скрипта возможно задать путь до собственного файла с настройками: python log_analyzer.py --config "путь"
